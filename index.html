HTML

  • HTML: Hypertext Markup Language.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !DOCTYPE: tells the browser what document type to expect.
  • lang="": HTML attribute used to tell the browser the language of the page. Attributes define additional information about an element.
  • Metadata: data about a webpage that the browser uses to effectively serve the webpage to users.
  • Opening Tag and Closing Tag: content for visible HTML elements are placed between them. Opening tag represented by a tag name surrounded by angle brackets. Closed tag is the same thing but with a forward slash before the tag name.
  • ul,ol, and li: elements representing unordered lists, ordered lists, and list items.

CSS

  • Three ways to style webpages using CSS are inline CSS, internal CSS, and external CSS style sheets.
  • Inline CSS style sheets are written in the HTML code to style one particular page element only.
  • Internal CSS style sheets contain CSS rules in the header section of the HTML file to style multiple elements.
  • External CSS style sheets keep all the CSS rules in a seperate file that are linked to the HTML file in the header.
  • CSS rule are written as: Selector{Property: Value;} and you can apply the same rule to multiple selectors with commas.
  • To add style to certain elements but not all the elements you can use a class attribute as the selector and assigning the attribute to specific elements.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main: switches current branch to the main branch.
  • git pull origin main: pulls the remote main branch from github into my local main branch on my computer. Always use before sending a branch to be merged to main branch.
  • Git add -A: stages changes made in locale branch that is being worked on.
  • git commit -m: git commit commits the changes made to the local branch being worked on. -m adds a message about what is being committed.
  • git push origin branch-name: pushes up changes made in the local branch to the remote repository on Github.
  • cd folder name: navigate you forward in git bash to the folder specified.
  • cd ..: navigates you in git bash backwards one folder.
  • code .: used to open VS code for the current directory git bash is using.
  • touch folder name: used to create new file in the folder of the directory git bash is currently using.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Variables created by the keyword const are variables that are unchangable in the code.
  • Variables created by the keyword let are variables that can be changed in the code.
  • Array: A single variable used to hold a group of data.
  • X++: shortcut way of writing x+1.
  • Control flow is the order in which a computer executes code in a script.
  • =: Assignment operator used to assign specific data to a variable.
  • ===: Strict equality operator that checks if two values are equal and returns a Boolean result of either true or false.
  • console.log(): Method used to output a message to the web console.
  • The javaScript file is linked in the body element of HTML while CSS is lined in the head element of HTML.
  • To open Chrome DevTools either right click browser and click inspect or use CTRL+Shift+I.
  • If statement: Conditional statement that introduces decision-making into code.
  • For loop: Loop that uses a predictable pattern to perform a task multiple times when executed.
  • A function is a reusable block of code that performs a specific task when called.
  • Functions do not automatically execute when the javascript file is run but instead must be called by the functions unique name.
  • To create a function you use the function keyword followed by the functions unique name and then squiggle brackets for the functions task code.
  • A functions unique name is followed by parentheses().
